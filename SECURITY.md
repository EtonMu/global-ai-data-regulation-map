# Security policy

## Supported version

Security fixes are applied to the latest version on the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a vulnerability that could put users or deployments at risk. Use GitHub’s private vulnerability reporting feature once it is enabled for the repository, or contact a maintainer privately.

Include the affected version, reproduction steps, likely impact, and any suggested mitigation. Avoid accessing data or systems beyond what is necessary to demonstrate the issue.

The current MVP is a read-only public research interface. It does not collect user accounts, uploads, or personal information.
